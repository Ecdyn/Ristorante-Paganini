**Section 1: STICKY HEADER (per D-01, D-02, D-03, FOUND-04)**

Implement as an IIFE `(function initStickyHeader() { ... }())`:

1. Get elements: `const header = document.getElementById('site-header');` and `const sentinel = document.querySelector('.hero-sentinel');`
2. Guard clause: `if (!header || !sentinel) return;`
3. Track scroll direction with `let lastScrollY = window.scrollY; let ticking = false;`
4. Create IntersectionObserver on sentinel with `{ threshold: 0 }`:
   - When `entry.isIntersecting` is true (hero visible): `delete header.dataset.state;` (transparent state — CSS `.header:not([data-state])` applies)
   - When `entry.isIntersecting` is false (scrolled past hero): set initial `header.dataset.state = 'scrolled';` then start monitoring scroll direction
5. Scroll direction detection (only active after scrolling past hero):
   - Use a scroll event listener BUT only fire logic via `requestAnimationFrame` throttle (the sentinel IO handles the heavy lift — scroll listener is minimal)
   - `window.addEventListener('scroll', onScroll, { passive: true });`
   - In `onScroll`: if `ticking` return; set `ticking = true`; call `requestAnimationFrame(() => { ... ticking = false; })`
   - Inside rAF: `const currentY = window.scrollY;` — if sentinel is NOT intersecting (past hero): if `currentY > lastScrollY + 5` (scrolling down with 5px dead zone): `header.dataset.state = 'hidden';` else if `currentY < lastScrollY - 5` (scrolling up): `header.dataset.state = 'scrolled';` — update `lastScrollY = currentY;`
   - If sentinel IS intersecting (back at top): do nothing (IO callback handles it)
6. Store the observer reference and a `pastHero` boolean toggled by the IO callback so the scroll listener knows when to activate direction detection.

**Section 2: MOBILE MENU (per D-04, D-05, D-06, FOUND-05)**

Implement as an IIFE `(function initMobileMenu() { ... }())`:

1. Get elements: `const toggle = document.getElementById('menu-toggle');` and `const menu = document.getElementById('mobile-menu');` and `const closeBtn = menu?.querySelector('.mobile-menu-close');`
2. Guard clause: `if (!toggle || !menu) return;`
3. Define `const FOCUSABLE = 'a[href], button:not([disabled])';`
4. `openMenu()` function:
   - `toggle.setAttribute('aria-expanded', 'true');`
   - `menu.dataset.open = 'true';`
   - `document.body.style.overflow = 'hidden';`
   - Focus first focusable: `const first = menu.querySelector(FOCUSABLE); if (first) first.focus();`
5. `closeMenu()` function:
   - `toggle.setAttribute('aria-expanded', 'false');`
   - `delete menu.dataset.open;`
   - `document.body.style.overflow = '';`
   - `toggle.focus();` (return focus to trigger)
6. Event listeners:
   - `toggle.addEventListener('click', () => { toggle.getAttribute('aria-expanded') === 'true' ? closeMenu() : openMenu(); });`
   - `if (closeBtn) closeBtn.addEventListener('click', closeMenu);`
   - Anchor auto-close (per D-05): `menu.querySelectorAll('a[href^="#"]').forEach(link => { link.addEventListener('click', closeMenu); });`
   - Escape key: `menu.addEventListener('keydown', (e) => { if (e.key === 'Escape') closeMenu(); });`
7. Focus trap (Tab cycling):
   - `menu.addEventListener('keydown', (e) => { if (e.key !== 'Tab') return; const focusable = [...menu.querySelectorAll(FOCUSABLE)]; if (!focusable.length) return; const first = focusable[0]; const last = focusable[focusable.length - 1]; if (e.shiftKey && document.activeElement === first) { e.preventDefault(); last.focus(); } else if (!e.shiftKey && document.activeElement === last) { e.preventDefault(); first.focus(); } });`
