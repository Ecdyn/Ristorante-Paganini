  <action>
Create 4 favicon files in the `assets/` root directory:

**1. assets/favicon.svg** — Scalable vector favicon:
```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <rect width="32" height="32" rx="4" fill="#FBF5F5"/>
  <text x="16" y="24" font-family="Georgia, 'Times New Roman', serif" font-size="22" font-weight="400"
        fill="#7A2D3A" text-anchor="middle">P</text>
</svg>
```
The `rx="4"` adds subtle rounded corners. Use Georgia serif fallback (not Cormorant Garamond — webfonts don't load in favicon context per research Pitfall 7). The "P" is positioned at y="24" for optical vertical centering of the letterform within the 32x32 viewbox.

**2. assets/favicon.ico** — 32x32 ICO fallback:
Generate from favicon.svg. Approach options (in order of preference):
- If ImageMagick is available: `magick assets/favicon.svg -resize 32x32 assets/favicon.ico`
- If ImageMagick is not available: use a programmatic approach — create a minimal 32x32 ICO file. The ICO format for a single 32x32 image is: 6-byte header + 16-byte directory entry + 32x32 BMP or PNG payload.
- Simplest fallback: create a 32x32 PNG first (cream background #FBF5F5, wine "P" #7A2D3A centered), then wrap in ICO container, OR use the SVG as the primary favicon and note in the SUMMARY that ICO can be regenerated with any SVG-to-ICO tool before launch.

**3. assets/apple-touch-icon.png** — 180x180 PNG:
Generate from the same design as favicon.svg but at 180x180 resolution.
- Cream background (#FBF5F5) filling full 180x180 canvas
- Wine-colored "P" (#7A2D3A) centered, approximately 120px font size equivalent
- If ImageMagick available: `magick -size 180x180 xc:#FBF5F5 -font Georgia -pointsize 120 -fill "#7A2D3A" -gravity center -annotate 0 "P" assets/apple-touch-icon.png`
- If no image tool available: create a minimal valid PNG programmatically or note as a manual generation step

**4. assets/site.webmanifest** — exact content:
```json
{
  "name": "Ristorante Paganini",
  "short_name": "Paganini",
  "icons": [
    { "src": "apple-touch-icon.png", "sizes": "180x180", "type": "image/png" }
  ],
  "theme_color": "#7A2D3A",
  "background_color": "#7A2D3A",
  "display": "browser"
}
```

The webmanifest `theme_color` and `background_color` are both #7A2D3A (wine accent) per D-12. The `display: "browser"` is correct — this is not a PWA, just providing browser chrome tinting on Android.
