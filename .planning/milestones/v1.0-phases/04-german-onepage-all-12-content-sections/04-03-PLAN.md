The lightbox IIFE must implement:

1. **DOM hooks** (guard-clause pattern like existing IIFEs):
   ```javascript
   const lightbox = document.getElementById('lightbox');
   const lightboxImg = document.getElementById('lightbox-img');
   const prevBtn = document.getElementById('lightbox-prev');
   const nextBtn = document.getElementById('lightbox-next');
   const closeBtn = document.getElementById('lightbox-close');
   const triggers = document.querySelectorAll('[data-lightbox-trigger]');
   if (!lightbox || !lightboxImg || !triggers.length) return;
   ```

2. **State tracking**:
   - `let currentIndex = 0;` — tracks which image is shown
   - `const images = [...triggers]` — array of all trigger elements
   - `let lastFocused = null;` — stores the element that opened the lightbox

3. **Open function** `function openLightbox(index)`:
   - Set `currentIndex = index`
   - Set `lightboxImg.src = images[index].getAttribute('data-lightbox-src')` (the full-size image path)
   - Set `lightboxImg.alt = images[index].getAttribute('data-lightbox-alt')` (accessible alt text)
   - Set `lastFocused = document.activeElement`
   - Set `lightbox.dataset.open = 'true'` (mirrors mobile menu pattern)
   - Set `lightbox.setAttribute('aria-hidden', 'false')`
   - Set `document.body.style.overflow = 'hidden'`
   - Focus the close button: `closeBtn.focus()`
   - Update prev/next button disabled state based on index

4. **Close function** `function closeLightbox()`:
   - `delete lightbox.dataset.open`
   - `lightbox.setAttribute('aria-hidden', 'true')`
   - `document.body.style.overflow = ''`
   - `if (lastFocused) lastFocused.focus()` — return focus to trigger

5. **Navigate function** `function navigate(direction)`:
   - Calculate `newIndex = currentIndex + direction`
   - If `newIndex >= 0 && newIndex < images.length`, call `openLightbox(newIndex)`

6. **Event listeners**:
   - Each trigger: `click` event calls `openLightbox(index)` with `e.preventDefault()`
   - Close button: `click` calls `closeLightbox()`
   - Prev button: `click` calls `navigate(-1)`
   - Next button: `click` calls `navigate(1)`
   - Lightbox backdrop: `click` on lightbox element itself (not children) calls `closeLightbox()`

7. **Keyboard handling** (on lightbox element):
   - `Escape` → `closeLightbox()`
   - `ArrowLeft` → `navigate(-1)`
   - `ArrowRight` → `navigate(1)`
   - `Tab` → focus trap cycling between closeBtn, prevBtn, nextBtn (same pattern as mobile menu focus trap)

8. **Focus trap** (mirror mobile menu implementation):
   ```javascript
   const FOCUSABLE = 'button:not([disabled])';
   // ... Tab key handler cycling between first and last focusable
   ```

9. **Reduced motion check**:
   ```javascript
   const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
   ```
   If reduced motion preferred, add/remove `data-open` without transition (CSS handles this via existing `@media (prefers-reduced-motion: reduce)` rule — JS just needs to not interfere).

10. **ARIA attributes on lightbox element** (set in JS because the element may be in HTML without these):
    - `role="dialog"`
    - `aria-modal="true"`
    - `aria-label="Bildergalerie"`
    - `aria-hidden="true"` (default, toggled by open/close)
