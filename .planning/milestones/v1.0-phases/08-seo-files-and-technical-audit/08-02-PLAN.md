  <action>
Create the directory tools/ at the project root, then create tools/validate.js using ONLY Node.js built-in modules fs and path (per D-11). No npm install, no external dependencies.

The script must do the following (per D-12):

1. Define ROOT as path.join(__dirname, '..') since the script lives in tools/ one level below project root.

2. Define FILES_TO_VALIDATE array with 3 paths:
   - path.join(ROOT, 'de', 'index.html')
   - path.join(ROOT, 'en', 'index.html')
   - path.join(ROOT, 'it', 'index.html')

3. Define JSONLD_REGEX to extract script blocks of type application/ld+json:
   Pattern: /<script\s+type="application\/ld\+json">([\s\S]*?)<\/script>/gi

4. Define REQUIRED_FIELDS object mapping @type to required field arrays:
   - Restaurant: ['name', 'address', 'telephone', 'openingHoursSpecification', 'servesCuisine']
   - LocalBusiness: ['name', 'address', 'telephone', 'openingHoursSpecification', 'servesCuisine']
   - FAQPage: ['mainEntity']

5. For each file in FILES_TO_VALIDATE:
   a. Check fs.existsSync first -- if file missing, print error with full path and increment error count
   b. Read file with fs.readFileSync(filePath, 'utf8')
   c. Reset JSONLD_REGEX.lastIndex = 0 before each file
   d. Extract all JSON-LD blocks using regex exec loop
   e. Parse each block with JSON.parse wrapped in try/catch -- on parse failure, print "FAIL [relative_path] Invalid JSON: error.message" and increment errors
   f. Handle @type as both string and array form: normalize with [].concat(block['@type'] || [])
   g. For each recognized type, check every required field exists and is truthy
   h. Print "PASS [relative_path] @type=X field: Y" for each passing check
   i. Print "FAIL [relative_path] @type=X missing field: Y" for each failing check

6. Print summary at the end:
   - "--- Validation Summary ---"
   - "Passed checks: N"
   - "Failed checks: N"
   - If zero errors: "ALL JSON-LD VALID" and process.exit(0)
   - If any errors: "VALIDATION FAILED -- fix errors above before launch" and process.exit(1)

IMPORTANT: Use function syntax (not arrow functions) for compatibility. Use var or const, not let (broader Node.js compatibility). Use console.log for passes, console.error for failures.
