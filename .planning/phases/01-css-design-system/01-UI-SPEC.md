## Spacing Scale

Declared values (multiples of 4px, using `rem` in implementation):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px (0.25rem) | Icon gaps, fine-grained inline padding |
| `--space-2` | 8px (0.5rem) | Compact element spacing, button padding vertical |
| `--space-3` | 12px (0.75rem) | Input padding, tag spacing |
| `--space-4` | 16px (1rem) | Default element spacing, button padding horizontal |
| `--space-6` | 24px (1.5rem) | Card padding, nav item gaps |
| `--space-8` | 32px (2rem) | Component gaps, form field stacks |
| `--space-12` | 48px (3rem) | Sub-section spacing |
| `--space-16` | 64px (4rem) | Large component padding |
| `--space-20` | 80px (5rem) | Section vertical spacing — minimum (D-08) |
| `--space-24` | 96px (6rem) | Section vertical spacing — mid |
| `--space-30` | 120px (7.5rem) | Section vertical spacing — maximum (D-08) |

Section spacing uses fluid clamping: `clamp(var(--space-20), 8vw, var(--space-30))`.

Exceptions:
- Touch targets (buttons, nav links): minimum 44px height — enforced via `min-height: 44px` in `.btn` block and nav link padding, not a spacing token.
- Hero section: full viewport height (`100vh`) — layout value, not a spacing token.
- Header height: 72px (4.5rem) — stored as `--header-height` for `scroll-padding-top` use in Phase 2.
- `--space-3` (12px): 12 is a multiple of 4 and sits between 8px and 16px on the standard scale. It is justified as an explicit scale step for compact interactive elements (input padding, pill tags) where 8px is too tight and 16px is too loose. It is used only in those two specific contexts and not as a general spacing unit.

Source: CONTEXT.md D-08, D-09; RESEARCH.md Pattern 2 token architecture.

## Typography

### Fonts

| Role | Family | Format | Weights Loaded |
|------|--------|--------|----------------|
| Heading (H1–H4) | Cormorant Garamond | woff2, self-hosted | 400, 400i |
| Body, nav, buttons, captions | Lato | woff2, self-hosted | 400, 700 |

Font loading: `font-display: swap` on all `@font-face` declarations.
Font source: google-webfonts-helper (gwfh.mranftl.com), Latin subset only.
No request to fonts.googleapis.com or fonts.gstatic.com at any point.

Cormorant Garamond weight reduction: 300 (light) and 600 (semibold) are dropped. Weight 400 carries all heading roles. Visual hierarchy is established via fluid size scaling, uppercase + letter-spacing on H2, and line-height — not weight variation within the serif family.

### Type Scale

Four named font-size tokens. Body base (1rem / 16px) is set on `html` and `body` as the cascade default — it is not a named scale token.

| Role | Token | Size | Weight Token | Line Height | Font Family |
|------|-------|------|-------------|-------------|-------------|
| Hero display + H1 | `--font-size-display` | `clamp(2.5rem, 6vw, 4.5rem)` — fluid | `--font-weight-normal` (400) | 1.15 | Cormorant Garamond |
| H2 section title | `--font-size-heading-lg` | `clamp(1.625rem, 3vw, 2rem)` — fluid | `--font-weight-normal` (400) | 1.2 | Cormorant Garamond |
| H3 subsection | `--font-size-heading-sm` | 1.375rem (22px) | `--font-weight-normal` (400) | 1.25 | Cormorant Garamond |
| Caption / legal | `--font-size-sm` | 0.875rem (14px) | `--font-weight-normal` (400) | 1.5 | Lato |

Body copy (unlisted token): 1rem (16px), `--font-weight-normal` (400), line-height 1.6, Lato. Lead/intro paragraphs share this size and are differentiated from body by `font-style: italic` or a slightly increased line-height (1.7), not a distinct size token.

Mobile behavior: `--font-size-display` and `--font-size-heading-lg` use fluid `clamp()` values and shrink automatically with viewport. `--font-size-heading-sm` stays fixed at 1.375rem on all viewports. Body and caption sizes do not change across breakpoints.

### Font Weight Tokens

Exactly two weight tokens. No other weight values appear anywhere in `main.css`.

| Token | Value | Applied To |
|-------|-------|-----------|
| `--font-weight-normal` | 400 | Cormorant Garamond (all headings), Lato body, captions, nav links |
| `--font-weight-bold` | 700 | Lato buttons, Lato bold emphasis, nav brand name |

### H2 Section Title Treatment (Claude's Discretion — resolved)

H2 section titles use: uppercase (`text-transform: uppercase`) + tracked letter-spacing (`letter-spacing: 0.12em`) in Cormorant Garamond weight 400. Uppercase + generous tracking creates visual distinction without relying on weight variation. A short decorative rule (2px wine-colored `::after` pseudo-element below the H2) adds section identity. This treatment is premium and reads as refined rather than loud.

Source: CONTEXT.md D-05 (heading scale), RESEARCH.md Pattern 2 token block.

### 60/30/10 Split

| Role | Hex | Token | Usage |
|------|-----|-------|-------|
| Dominant (60%) | `#FBF5F5` | `--color-surface` | Primary page background — all main sections |
| Secondary (30%) | `#F5EDEE` | `--color-surface-alt` | Alternating sections (about, FAQ, contact), footer pre-border |

| Dark surface | `#2C1A1D` | `--color-surface-dark` | Footer background, hero overlay base color |

### Section Separation (Claude's Discretion — resolved)

Method: alternating background tints between `--color-surface` and `--color-surface-alt` plus generous whitespace. No decorative divider lines between sections. The footer uses `--color-surface-dark` (dark espresso) as the sole dark section on the page — this creates a strong visual anchor at the bottom while keeping the rest of the site airy and light.

### Accent Reserved For (explicit list)

Accent color (`#7A2D3A`) is ONLY used on:
1. Primary CTA buttons (solid fill with white text — D-03)
2. Text links and inline anchor underlines
3. Navigation active/current page indicator
4. Focus-visible ring outline on interactive elements
5. Section label decorative rule (short `::after` element under H2)
6. Icon strokes on the contact section (phone, email)

Accent is NOT used on: body text, background surfaces, borders, shadows, captions, or decorative backgrounds.

## Interaction Tokens

Phase 1 defines interaction tokens as CSS custom properties. Actual interactive behaviors (sticky header, mobile menu) are Phase 2. The tokens establish the contract.

### Transitions

| Token | Value | Applied To |
|-------|-------|-----------|
| `--transition-fast` | 150ms ease | Focus rings, hover color changes |
| `--transition-base` | 250ms ease | Button hover, link hover, nav state |
| `--transition-slow` | 400ms ease | Header scroll state, section reveals |

### Focus Indicator (Claude's Discretion — resolved)

All interactive elements use `:focus-visible` (not `:focus`) to avoid mouse-click outlines. Style: `outline: 3px solid var(--color-accent)` with `outline-offset: 3px`. This satisfies WCAG 2.1 AA 3:1 minimum for non-text UI components. The wine red on cream background passes this ratio.

`prefers-reduced-motion` media query wraps all transition and animation declarations. When motion is reduced, transitions collapse to `duration: 0.01ms`.

### Shadows and Radii

| Token | Value | Usage |
|-------|-------|-------|
| `--shadow-sm` | `0 1px 3px rgba(42, 31, 32, 0.08)` | Card lift, subtle depth |
| `--shadow-md` | `0 4px 12px rgba(42, 31, 32, 0.12)` | Header on scroll, modal |
| `--shadow-lg` | `0 8px 32px rgba(42, 31, 32, 0.16)` | Large card emphasis |
| `--radius-sm` | 4px | Tags, pills |
| `--radius-md` | 8px | Buttons, cards |
| `--radius-lg` | 16px | Large container elements |

## CUBE CSS File Structure

The single `assets/css/main.css` file is organized into six named layers in this exact order:

```
1. DESIGN TOKENS    — :root CSS custom properties (all tokens above)
2. GLOBAL / RESET   — box-sizing, margin resets, base element styles
3. COMPOSITION      — layout primitives (.flow, .cluster, .wrapper, .grid)
4. UTILITIES        — single-purpose classes (.visually-hidden, .text-center)
5. BLOCKS           — component styles (.header, .hero, .btn, .section, etc.)
6. EXCEPTIONS       — data-attribute overrides ([data-state="scrolled"])
```

Each layer is separated by a block comment header. A table of contents appears in the file header.

### Editable Content Zone Comment Convention (FOUND-10)

All HTML files across all phases must mark live-data regions with these exact comment patterns:

```html
<!-- EDITABLE: [description of what to update] -->
<!-- SYNC: Update this value in de/, en/, and it/ files -->
<!-- PLACEHOLDER: Replace with real content before launch -->
```

This convention is documented in the `main.css` file header so any agency editor opening any file in the project sees the standard immediately.
