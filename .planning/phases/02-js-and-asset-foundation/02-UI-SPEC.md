## Interaction Behavior Contract

### Sticky Header (FOUND-04, D-01, D-02, D-03)

**Mechanism:** IntersectionObserver on a sentinel element placed at the bottom of the hero section.
- When sentinel exits viewport (scrolled past hero): add `data-scrolled="true"` to `<header>`
- When sentinel re-enters viewport (scrolled back to top): remove `data-scrolled` attribute
- CSS selects on `[data-scrolled]` to apply solid background state
- No scroll event listeners — zero main-thread blocking

**Scroll threshold:** Sentinel is a 1px div positioned at the bottom of the hero section. Intersection ratio threshold: 0 (fires as soon as sentinel leaves viewport).

**Transition timing:** 400ms ease on background-color and box-shadow (--transition-slow). No transition on color to avoid flash-of-wrong-color on page load.

**Expected states:**
| State | Trigger | Visual |
|-------|---------|--------|
| Transparent | Page at top (hero visible) | background: transparent, no shadow |
| Solid | Scrolled past hero | background: --color-surface, shadow: --shadow-sm |

### Mobile Menu (FOUND-05, D-04, D-05, D-06)

**Trigger:** Hamburger button `<button type="button" id="menu-toggle" aria-expanded="false" aria-controls="mobile-menu">`

**Open behavior:**
- Set `aria-expanded="true"` on button
- Set `data-open="true"` on overlay container (`<div id="mobile-menu" role="dialog" aria-modal="true" aria-label="Navigation">`)
- CSS: overlay transitions from `visibility: hidden; opacity: 0` to `visibility: visible; opacity: 1` using --transition-base (250ms ease)
- Move focus to first nav link inside overlay (focus management for keyboard users)
- Prevent body scroll: add `overflow: hidden` to `<body>` while open

**Close behavior (3 triggers):**
1. Close button tap/click
2. Any anchor link tap/click within overlay
3. Escape key press

On close:
- Set `aria-expanded="false"` on button
- Remove `data-open` attribute from overlay
- CSS: overlay transitions back to hidden state
- Restore body scroll
- Return focus to hamburger button

**Keyboard navigation inside overlay:**
- Tab cycles through nav links and close button only (focus trap)
- Escape key closes overlay from any focused element

**Hamburger icon design:**
- Three horizontal bars, 24px wide, 2px tall, 5px gap between bars
- Drawn with CSS (no SVG, no image): three `<span>` elements or CSS `::before` / `::after` + single `<span>`
- No animated morph-to-X on open — three bars simply hide, close button (text or static X icon) appears separately

### Smooth Scroll (FOUND-06)

**Mechanism:** CSS `scroll-behavior: smooth` on `<html>` element — zero JS required.
- Applied in main.css (Phase 1 or Phase 2 addition to global styles)
- `scroll-padding-top: var(--header-height)` on `<html>` element — ensures sticky header does not obscure anchored section headings

**Reduced motion:** `@media (prefers-reduced-motion: reduce)` — set `scroll-behavior: auto` to override smooth scroll for users who have opted out.

**Expected behavior:** Any `<a href="#section-id">` link causes viewport to scroll smoothly to `#section-id`, landing with the section heading fully visible below the 72px sticky header.

### Placeholder Images (D-07, D-08, D-09)

**Format:** SVG placeholder files (inline-able, zero external dependencies, clearly show dimensions and section name)

**Naming convention:** `{section}-{variant}.{format}` — examples:
- `hero-desktop.avif` / `hero-desktop.webp` / `hero-desktop.jpg`
- `hero-mobile.avif` / `hero-mobile.webp` / `hero-mobile.jpg`
- `gallery-01.avif` / `gallery-01.webp` / `gallery-01.jpg`
- `about-bg.avif` / `about-bg.webp` / `about-bg.jpg`
- `og-image.jpg` (OG/social sharing image, JPEG only, 1200x630)

**Placeholder visual style:**
- Background fill: --color-surface-alt (#F5EDEE)
- Text label: dimensions + section name, --color-text-muted (#6B5557), Lato 14px
- Border: 1px dashed --color-border (#DFD0D2)
- Comment in HTML: `<!-- PLACEHOLDER: Replace with real photography before launch -->`

**Required image sets for Phase 2:**

| Section | Dimensions | Variants |
|---------|-----------|---------|
| Hero desktop | 1400x800 | avif, webp, jpg |
| Hero mobile | 768x500 | avif, webp, jpg |
| Gallery (x6) | 800x600 | avif, webp, jpg |
| About background | 1200x600 | avif, webp, jpg |
| OG image | 1200x630 | jpg only |

**picture element pattern (D-09):**
```html
<picture>
  <source srcset="assets/img/hero-desktop.avif" type="image/avif" media="(min-width: 768px)">
  <source srcset="assets/img/hero-desktop.webp" type="image/webp" media="(min-width: 768px)">
  <source srcset="assets/img/hero-mobile.avif" type="image/avif">
  <source srcset="assets/img/hero-mobile.webp" type="image/webp">
  <img src="assets/img/hero-desktop.jpg" alt="" width="1400" height="800"
       loading="eager" fetchpriority="high">
</picture>
```
All `<img>` elements must carry explicit `width` and `height` attributes. Hero uses `loading="eager" fetchpriority="high"`. All below-fold images use `loading="lazy" decoding="async"`.

## Implementation Notes for Executor

1. **JS state management via data attributes:** JS toggles `data-scrolled` on `<header>` and `data-open` on the overlay `<div>`. CSS selects on these attributes. Do not toggle class names — CUBE CSS convention uses data attributes for exceptions and state (02-CONTEXT.md code context).

2. **Single JS file, no inline scripts:** All behaviors in `assets/js/main.js` with `defer`. Zero `<script>` tags in `<body>`. Zero inline event handlers (`onclick=` etc.).

3. **CSS scroll additions:** `scroll-behavior: smooth` and `scroll-padding-top: var(--header-height)` on `<html>` — add to main.css global section if not already present from Phase 1.

4. **IntersectionObserver sentinel placement:** A `<div class="hero-sentinel" aria-hidden="true">` placed as the last child of the hero section, styled to zero dimensions in CSS. JS observes this element.

5. **Focus trap implementation:** When mobile overlay is open, Tab and Shift+Tab must cycle only within the overlay. Implement with a focusable-elements selector and manual key event handler.

6. **Placeholder images as SVG:** Generate SVG files programmatically — they require no external tools, load instantly, and can be committed to the repo as-is. Real AVIF/WebP/JPEG sets are a post-launch task (POST-01).

7. **Path convention:** All asset references in HTML use relative paths from the page's own directory (e.g., `../assets/js/main.js` from `/de/index.html`). This is consistent across all language dirs.
