# Phase 5 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Phase character note:** Phase 5 is a pure translation and localization task. No new layouts, components, CSS, or JS are introduced. All design decisions below are inherited directly from the existing `assets/css/main.css` design system and `de/index.html` implementation. ## hreflang & Head Metadata Contract

Both new pages must carry the full reciprocal hreflang cluster (CONTEXT.md D-14, REQUIREMENTS.md SEO-03):

```html
<link rel="alternate" hreflang="de"        href="https://www.example.com/de/">
<link rel="alternate" hreflang="en"        href="https://www.example.com/en/">
<link rel="alternate" hreflang="it"        href="https://www.example.com/it/">
<link rel="alternate" hreflang="x-default" href="https://www.example.com/de/">
```

Self-canonical per page:
- `en/index.html`: `<link rel="canonical" href="https://www.example.com/en/">`
- `it/index.html`: `<link rel="canonical" href="https://www.example.com/it/">`

`de/index.html` already has the correct hreflang cluster — verify all four return tags are present before closing Phase 5.

Footer language switcher: All three footers must link correctly to each other. The `aria-current="page"` (or equivalent active class) must be set on the link matching the current page's language.
